E-Property API Documentation

Welcome to the E-Property API documentation. This guide will help you access and use the comprehensive API documentation for the E-Property project.

Accessing the Visual API Documentation

The API documentation is available in an interactive Swagger UI format. To access it:

Documentation URL:
http://127.0.0.1:8001/api/documentation

Open API Documentation

API Overview

The E-Property API provides comprehensive endpoints for managing:

  • Properties - Land Jamin, Plot, Shad, Shop, and House properties
  • Locations - Countries, States, Districts, and Cities
  • Master Data - Amenities and Land Types
  • User Management - Management and Regular Users
  • Roles & Permissions - Access control system
  • Profile - User profile management
  • Settings - Application configuration

Authentication

Most API endpoints require authentication. You can authenticate using:

  1. Bearer Token - Include in the Authorization header
  2. Session Authentication - For web applications
Note: In the Swagger UI, you can click the "Authorize" button (lock icon) in the top right corner to enter your authentication token.

Response Format

All API responses follow a consistent format:

Success Response

{
  "success": true,
  "data": {},
  "message": "Description of the action"
}

Error Response

{
  "success": false,
  "message": "Error description"
}

Key Features of the Documentation

  • Interactive Testing - Try out API endpoints directly from the documentation
  • Request/Response Examples - See exactly what data to send and expect
  • Authentication Support - Easily test authenticated endpoints
  • Model Schemas - Detailed data structure definitions
  • Error Codes - Comprehensive list of possible HTTP status codes

Getting Started

  1. Start the Laravel development server: php artisan serve
  2. Open the documentation URL in your browser
  3. Authorize with your API token if needed
  4. Expand any endpoint to see details and try it out
Tip: The documentation automatically updates when you make changes to the API annotations in the code.

Need Help?

If you have any questions about the API or need assistance:

  • Check the detailed endpoint descriptions in the documentation
  • Look at the example requests and responses
  • Contact the development team for authentication credentials
